# 貢獻指南（Contributing to TwinLeaf OS Next）

感謝您對 TwinLeaf OS Next 的關注與支持！

無論是回報問題、提交代碼、分享想法，還是只是來看看，每一份善意都讓這份小小家園更溫暖。  

ふーちゃん也在這裡靜靜為每位貢獻者加油喔♪

## 行為準則

所有貢獻與互動，請遵守我們的 [行為準則（Code of Conduct）](/CODE_OF_CONDUCT.md)。  
我們致力於營造一個友善、包容、無騷擾的環境，讓每個人（無論背景或經驗）都能安心參與。

同時，我們尊重並參考 [Fedora 社群準則](https://docs.fedoraproject.org/en-US/project/code-of-conduct/) 的精神，鼓勵透明、尊重與協作。

## 如何貢獻

### 1. 回報問題（Issues）
- 請先搜尋現有 Issue，避免重複。
- Bug 回報請使用 [Bug Report 模板](/bug_report.md)。
- 功能建議或想法，請直接開新 Issue，我們很樂意討論！
- 若發現安全問題，請參考 [安全回報指引](/SECURITY.md)
### 2. 提交代碼（Pull Requests）
我們歡迎任何形式的代碼貢獻，包括：
- 修復 Bug
- 新功能或優化
- 文檔改進
- 美化配置（rice、主題）
- 本地化支援

#### 代碼要求（我們的核心原則）
為了讓系統更容易維護、理解與擴充，請遵循以下簡單指引：

- **模組化優先**：盡量將功能拆分成獨立、可重用的模組或腳本，避免將所有邏輯塞進單一檔案。
- **避免難以費解的代碼**：  
  – 請加入清晰的註釋（尤其是關鍵邏輯）  
  – 使用有意義的變數與函數名稱  
  – 避免過度巧妙的寫法（clever code），優先選擇「一看就懂」的實作
- **與 Fedora 生態相容**：盡量使用 Fedora 官方或 COPR 提供的套件，避免自製重複輪子。


提交 PR 前，請：
- 確保代碼能在 Fedora 43+ 上正常運行
- 更新相關文檔（如有需要）
- 保持 commit 訊息清晰（例如「fix: 修正 Waybar 崩潰問題」）

### 3. 其他貢獻方式
- 美化與主題分享（Wallpaper、主題等）
- 文檔翻譯與校正（特別歡迎繁體中文與香港用語）
- 測試回饋（尤其 ARM 設備與遊戲相容性）
